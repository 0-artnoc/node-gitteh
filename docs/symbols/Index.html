Class Detail

Index()
The Index represents the cache Git stores for your working directory. When you "stage" a file in Git, this is basically just a command that will add or update your file in the Git index. Running a Commit in Git is essentially just writing a Commit with a Tree that matches an Index. Note: There will be a method soon to create a Tree from an opened Index, this feature has not yet been implemented by libgit2, as soon as it is it will be in Gitteh.
See:
Repository#getIndex

Method Detail

  • {Boolean} addEntry(fileName, callback)
    Adds an entry to the Index for the given fileName. This file should already exist in the working directory of the Repository. Note: this function will fail if called on a Index for a bare repository, as it has no working directory.
    Parameters:
    {String} fileName
    Name of the file to update index entry for.
    {Function} callback Optional
    When provided, the callback will be notified when the entry has been added asynchronously.
    Throws:
    If there was an error adding the entry.
    Returns:
    {Boolean} true if successful
  • {IndexEntry} findEntry(fileName, callback)
    Searches the Index for an entry that matches the given filename.
    Parameters:
    {String} fileName Optional
    The filename of entry to search for.
    {Function} callback Optional
    If provided, entry will be searched for in a non-blocking manner, and results returned to the callback.
    Throws:
    If entry could not be found, or some other error occurs.
    Returns:
    {IndexEntry}
  • {IndexEntry} getEntry(index, callback)
    Retrieves an IndexEntry from the Index.
    Parameters:
    {Integer} index
    The index to retrieve.
    {Function} callback Optional
    When provided, IndexEntry will be retrieved asynchronously and provided back to the callback.
    Throws:
    If index is out of bounds, or some other error occurs.
    Returns:
    {IndexEntry}
  • {Boolean} write(callback)
    Changes to an Index will not be updated on the actual index file until this method is called.
    Parameters:
    {Function} callback Optional
    When provided, the writeback will be performed in an asynchronous fashion, and the result will be passed back to the callback.
    Throws:
    If there was an error writing the Index.
    Returns:
    {Boolean} true if successful.
